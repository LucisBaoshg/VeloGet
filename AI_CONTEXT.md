## 2. Known Issues & "Gotchas"

### 🚨 Flet Framework (v0.80+) - CRITICAL API CHANGES

**Issue**: `FilePicker` causes "Unknown control" or hangs if used incorrectly.
**History**: This issue has recurred 3+ times during refactoring. DO NOT REVERT TO OLD PATTERNS.

#### ❌ BROKEN / DEPRECATED PATTERNS (DO NOT USE):
1.  **Adding to Overlay**: `page.overlay.append(picker)` is **NO LONGER NEEDED** for imperative usage.
2.  **Callbacks**: Defining `on_result` in `__init__` is **OBSOLETE**.
3.  **Member Variables**: Do not store `self.picker = ft.FilePicker()` in classes if you only need imperative access.

#### ✅ CORRECT PATTERN (Imperative Async/Await):
Flet v0.80+ supports calling static methods directly without instantiating a control in the view tree.

```python
import flet as ft

# 1. Pick Files
files = await ft.FilePicker().pick_files(allow_multiple=True)

# 2. Get Directory
path = await ft.FilePicker().get_directory_path()

# 3. Save File
save_path = await ft.FilePicker().save_file()
```

**Rule**: Always prefer the `await ft.FilePicker().method()` syntax inside async handlers.

### 🚨 Build & Packaging (Flet/Flutter)

**Issue**: App bundle size is enormous (>2GB).

#### ✅ MANDATORY BUILD COMMAND:
Always use `--exclude` to prevent bloating the app.

```bash

    --project VeloGet \
    --product VeloGet \
    --org com.lucifer \

```
